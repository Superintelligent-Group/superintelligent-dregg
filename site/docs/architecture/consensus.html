Consensus (Morpheus DAG-BFT)

Pyana federations use Morpheus, a DAG-based BFT consensus protocol with adaptive modes. It tolerates up to f = (n-1)/3 Byzantine faults while achieving optimistic fast-path in the common case (all-correct).

Adaptive Modes

  • Fast path: When all validators are correct, consensus commits in a single round-trip. No leader bottleneck.
  • Slow path: Under Byzantine behavior, falls back to a view-change protocol that guarantees safety and liveness.

DAG Structure

Each validator produces blocks that reference multiple parents (DAG edges). The causal ordering of the DAG determines the committed prefix without requiring explicit block proposal ordering.

Attested Roots

At each checkpoint interval, the federation collectively signs a state root. This "attested root" is the anchor against which STARK proofs are verified. Clients need only the latest attested root to verify any authorization proof.

Epoch Reconfiguration

The validator set can change at epoch boundaries. New validators join by having their public keys added to the epoch configuration. Departing validators are removed at the next epoch.

Detailed consensus protocol description, safety/liveness proofs, and performance analysis are being written.
