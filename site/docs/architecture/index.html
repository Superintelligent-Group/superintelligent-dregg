Architecture

Pyana is built as a layered system where each layer has clean boundaries and can be understood independently. The system spans ~125k lines of Rust across 22+ crates, with rigorous separation between cryptographic primitives, consensus, execution semantics, and user-facing APIs.

Design Principles

  • Capability attenuation IS incrementally verifiable computation. Each delegation is a fold over a committed fact set. The STARK proves a valid chain.
  • Offline-first verification. A STARK proof + attested root is complete. No federation liveness required.
  • Agents own their state. Receipt chains are portable proof-carrying state. Exit any federation at any time.
  • Progressive privacy. Three modes (trusted, selective, private) let developers choose the right tradeoff per interaction.
  • Post-quantum security. STARK proofs are hash-based (Poseidon2/BLAKE3). No elliptic curve assumptions on the verification path.
