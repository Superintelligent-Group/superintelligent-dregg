Federation Model

Pyana supports multiple federation topologies: single federations operating independently, cross-federation communication via bridge nodes, and multi-hop relay chains connecting distant federations.

Single Federation

The simplest deployment: N nodes (typically 4-7) run Morpheus consensus together, producing attested roots at regular intervals. All agents connect to nodes within this single federation.

Cross-Federation

Bridge nodes connect to two (or more) federations' gossip networks and relay:

  • Attested roots -- So agents in federation A can verify proofs from federation B
  • Revocations -- Token revocations propagate across boundaries
  • Receipts -- State proofs can reference cross-federation transitions
  • Conditional proofs -- Atomic cross-federation operations

Federation Exit

Agents own their state as receipt chains. To exit a federation, export the chain (or the IVC-compressed constant-size proof). Any verifier can check the chain's integrity without contacting the original federation. This is the anti-lock-in guarantee: your state is sovereign.

Detailed cross-federation protocol, trust model, and bridge security analysis are being written.
