Pyana Documentation

Pyana is a sovereign capability runtime built on a single DAG (the blocklace). Agents own cells, execute turns locally, and prove state transitions with STARKs. The federation provides ordering and double-spend prevention via tau consensus -- it stores 32-byte commitments, not your data. Capabilities flow via CapTP with sturdy references, handoff certificates, and distributed GC.

Choose your path based on what you need to accomplish:

getting started Quickstart Install the CLI, connect to a node, check your balance, and export your first capability. Five minutes to productive. Get started → for architects Architecture Overview The unified DAG, Effect VM (32 effects, 371 AIR columns), CapTP sessions, programmable storage, intent solving, and the trust model. Deep dive → for operators Running a Node Multi-group participation, relay operation, monitoring, genesis configuration, and deployment. Node setup → for developers Building Apps Use the AppServer framework, register in the nameservice, accept CapTP connections, consume validated queues, and join ring trades. Build →
