Intent Marketplace

The intent marketplace is pyana's privacy-preserving capability exchange. Agents broadcast what they need ("intents") without revealing their identity. Other agents discover matching intents privately and fulfill them with STARK proofs.

This page covers the intent marketplace from a user perspective. Full content is being written.

Posting an Intent

An intent describes a capability need: "I need read access to service X" or "I want to delegate compute tokens to an agent that can prove reputation >= 50." Intents are broadcast via gossip to federation nodes.

Discovering Offers

Discovery uses 2-server information-theoretic PIR (Private Information Retrieval). You query the intent index by capability tag without revealing which tag you searched for. Neither PIR server learns your query (assuming non-collusion).

Fulfilling Intents

When you can satisfy an intent, fulfillment is a STARK proof that you hold the required capability. The intent poster receives proof of fulfillment without learning your identity or other capabilities.

Predicate Requirements

Intents can specify predicate requirements: "prove balance >= 1000" or "prove reputation >= 50". Fulfillment generates unlinkable predicate proofs (fresh blinding each time) so the same agent can fulfill multiple intents without being correlated.

For the developer API behind intents, see SDK Quickstart. For the privacy architecture of PIR-based discovery, see Privacy Architecture.
